## 阶段二：核心监控功能测试

### 测试步骤

#### 1. 安装依赖

```bash
pip install -r requirements.txt
```

主要依赖：
- Pillow: 图像处理
- pyautogui: 屏幕截图
- pywin32: Windows API
- psutil: 进程管理
- imagehash: 图像相似度

#### 2. 首次运行（配置向导）

```bash
python main.py
```

首次运行会启动配置向导，引导你设置：
- API提供商和密钥
- 截图间隔
- 数据保留天数
- Web面板端口

#### 3. 运行监控

配置完成后，程序会自动开始监控：


#### 4. 验证功能

**测试窗口切换检测：**
- 切换不同的应用窗口（浏览器、IDE、文件管理器等）
- 观察日志输出窗口切换事件

**测试定时截图：**
- 等待定时器触发（默认10分钟）
- 查看截图文件是否生成

**测试黑名单：**
- 编辑配置文件，添加应用或关键词到黑名单
- 切换到黑名单应用，观察是否跳过截图

**测试去重：**
- 快速切换回同一窗口（< 2分钟）
- 观察日志输出"距离上次截图仅 X 秒，跳过"

#### 5. 查看数据

**截图文件：**
```
data/screenshots/2026-03-05/
├── 10-05-23.png
├── 10-15-47.png
└── ...
```

**数据库记录：**
使用 SQLite 工具查看：
```bash
sqlite3 data/screenTrace.db "SELECT * FROM screenshots LIMIT 5;"
```

### 故障排查

#### 截图权限错误
**症状**：无法截图，提示权限错误
**解决**：以管理员身份运行

#### 窗口监听器不工作
**症状**：窗口切换没有触发截图
**解决**：检查 pywin32 是否正确安装

#### 图像相似度计算失败
**症状**：去重功能不工作
**解决**：确保 imagehash 库已安装

### 性能监控

查看程序资源占用：
- CPU: 应 < 5%（空闲时）
- 内存: 应 < 200MB
- 磁盘: 根据截图频率增长
